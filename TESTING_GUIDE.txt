================================================================================

STEP 11: VERIFY IN CERTIFICATE GENERATION
================================================================================

To verify the mapped coordinates are used during generation:

1. Navigate to certificate generation page
2. Select the template you just mapped
3. Upload an Excel file with columns matching your field names
   - Example if you mapped "Student Name", "Event Date", "Recipient Role"
   - Excel file should have columns: "Student Name", "Event Date", "Recipient Role"
4. Generate certificates
5. Check generated PDFs/images:
   - Text appears at the coordinates you clicked
   - Text is positioned correctly relative to the template

If text appears in wrong positions:
  - Return to Template Mapper
  - Adjust field positions by removing and re-adding fields
  - Save again
  - Regenerate certificates

================================================================================
TROUBLESHOOTING
================================================================================

Issue: Template gallery empty
  └─ Check: backend/app/static/certificate_templates/ has PNG files
  └─ Ensure file extensions are .png (lowercase)
  └─ Restart backend server

Issue: Coordinates not captured on click
  └─ Check: Image fully loaded (no broken image icon)
  └─ Check: You added a field name first
  └─ Check: UI shows "Click on the certificate..." prompt
  └─ Try clicking directly on visible image area

Issue: Save fails with error
  └─ Check: All fields have names AND coordinates
  └─ Check: Network connectivity (F12 Dev Tools → Network tab)
  └─ Check: Backend is running (Terminal 1)
  └─ Check: Browser console for errors (F12)

Issue: Generated text in wrong position
  └─ Check: Field names match Excel column headers exactly (case-sensitive)
  └─ Check: X, Y values are reasonable (within image bounds)
  └─ Solution: Re-map the template with correct positions

Issue: Page crashes or freezes
  └─ Check: Browser console (F12 → Console tab) for errors
  └─ Check: Backend logs (Terminal 1) for API errors
  └─ Try: Hard refresh (Ctrl+Shift+R)
  └─ Try: Clear browser cache
