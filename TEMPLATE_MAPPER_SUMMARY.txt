================================================================================
API FLOW DIAGRAM
================================================================================

Frontend Request Flow:

1. USER SELECTS TEMPLATE
   ├─ Frontend GET /api/certificate-config/templates
   └─ Receives: ["template1.png", "template2.png", ...]

2. USER CLICKS TEMPLATE
   ├─ Frontend GET /api/certificate-config/config/template1.png
   └─ Receives: {} (empty if new) or {field: {x, y}, ...} (if exists)

3. USER ADDS FIELDS & CLICKS IMAGE
   ├─ Fields array built in React state
   ├─ X, Y auto-captured on image clicks
   └─ No backend calls during mapping (all in-memory)

4. USER SAVES CONFIGURATION
   ├─ Frontend POST /api/certificate-config/save
   ├─ Body: {template_name: "...", fields: [...]}
   └─ Backend: Save to MongoDB, return success

================================================================================
CERTIFICATE GENERATION INTEGRATION
================================================================================

When certificates are generated (png_generator.py):

Current Implementation:
  1. Load FieldPosition (percent-based presets)
  2. Use those coordinates to place text on PNG

New Integration (Already Updated):
  1. Check for UI-mapped config via get_template_config()
  2. If exists: Use pixel coordinates from config
  3. If not: Fall back to FieldPosition presets
  
This means:
  ✓ UI mappings override hardcoded presets
  ✓ Backward compatible with existing FieldPosition configs
  ✓ No breaking changes

HOW TO USE
================================================================================

1. START THE BACKEND
   cd backend
   python -m uvicorn app.main:app --reload

2. START THE FRONTEND
   cd frontend
   npm run dev

3. LOGIN AS SUPERADMIN
   - Navigate to Superadmin dashboard
   - Username: superadmin (from .env)
   - Password: superadmin (from .env)

4. CLICK "TEMPLATE VISUAL MAPPER"
   - Button in Superadmin dashboard header
   - Opens full-screen mapper interface

5. SELECT A CERTIFICATE TEMPLATE
   - Click any certificate thumbnail in the gallery

6. ADD YOUR FIRST FIELD
   - Click "Add New Field" button
   - Type field name (e.g., "Student Name", "Date", "Role")
   - Click "Confirm"
   - UI prompts: "Click on the certificate to set the position for '[Field]'"

7. CLICK ON THE CERTIFICATE
   - Click anywhere on the certificate image
   - X, Y coordinates are automatically captured
   - Field moves to "complete" status (shows green)

8. REPEAT FOR MORE FIELDS
   - Continue adding fields as needed
   - Each field can have unlimited fields

9. SAVE CONFIGURATION
   - Click "Save Configuration" button
   - Success message appears
   - Configuration stored in MongoDB

10. VERIFY IN GENERATION
    - When certificates are generated with this template
    - Fields will be placed at the coordinates you mapped
    - UI-mapped coordinates take priority over presets
